# 社区生鲜团购小程序-前端开发文档（毕设专用）

## 一、项目核心背景

本前端项目为**本科毕业设计（社区生鲜团购小程序）** 配套开发，基于UniApp（Vite+TypeScript）构建，仅适配微信小程序端，核心目标是实现“生鲜商品展示、拼团下单、AI食材推荐、团长管理、自提点查询、订单管理”六大核心功能，视觉风格与交互逻辑贴合“社区生鲜团购”场景，所有开发均服务于毕设演示与课题要求。

## 二、前端技术栈（固定不可变更）

| 模块     | 技术选型                     | 毕设适配说明                                                               |
| -------- | ---------------------------- | -------------------------------------------------------------------------- |
| 核心框架 | UniApp（Vite版）+ TypeScript | 统一前后端TS生态，毕设文档易体现“类型安全”，Vite提升开发/演示效率          |
| 样式方案 | Tailwind CSS 3.x             | 原子化CSS减少冗余代码，已预配置rpx单位适配小程序，主题色与课题视觉统一     |
| 状态管理 | Pinia（基础版）              | 简化版状态管理，仅存储用户信息、购物车、拼团临时数据，无需复杂模块化       |
| 路由管理 | UniApp原生路由API            | 用navigateTo/switchTab等原生API，避免自定义路由增加毕设复杂度              |
| 接口请求 | uni.request                  | 封装统一请求拦截器，适配后端RESTful接口，毕设演示可切换“模拟数据/真实接口” |
| 开发工具 | 微信开发者工具 + VSCode      | 本地调试/演示均在微信开发者工具中完成，无需跨端适配                        |

## 四、核心页面开发规范（按毕设优先级排序）

### 1. 首页（pages/index/index.vue）

#### 核心需求（贴合课题）

- 轮播图：展示生鲜秒杀/本地直供活动，点击跳转对应商品页；
- 分类入口：蔬菜/水果/肉蛋/水产4类，点击传参跳商品列表页；
- 热门拼团：展示4个核心商品，含拼团价、原价、“去拼团”按钮；
- AI食材助手入口：一键跳转，视觉用深绿#2F5233突出；
- 今日团长推荐：展示团长名称、覆盖小区、拼团人数（毕设加分项）。

#### 技术实现要求

- 所有样式用Tailwind，单位为rpx（已在tailwind.config.js配置）；
- 数据层：优先用模拟数据，联调阶段替换为/api/goods接口；
- 交互：按钮添加active-class点击反馈，下拉刷新加载最新数据。

### 2. 商品列表页（pages/goods/goods.vue）

#### 核心需求

- 接收首页分类参数（vegetable/fruit/meat/seafood），筛选对应商品；
- 支持价格排序/销量排序、关键词搜索；
- 商品列表：卡片式布局，含图片、名称（2行截断）、价格、销量、“加入拼团”按钮；
- 下拉加载更多、上拉回到顶部。

#### 技术实现要求

- 复用components/goods-card.vue组件，减少冗余；
- 搜索框用UniApp原生input，适配小程序样式；
- 排序逻辑：前端先模拟，联调后对接/api/goods（传sort参数）。

### 3. 拼团页（pages/group-buy/group-buy.vue）

#### 核心需求（课题核心功能）

- 接收商品ID，展示商品详情、拼团规则（2人/3人团、时效24小时）；
- 表单模块：选择拼团人数、填写收件人/电话/自提点；
- 核心按钮：“发起拼团”“加入他人拼团”；
- 拼团进度：展示已参与人数/目标人数，成团后弹窗提示。

#### 技术实现要求

- 表单校验：用utils/validate.ts校验手机号格式、必填项；
- 进度条：用Tailwind实现，主色#F08800填充进度；
- 数据层：模拟拼团状态，联调后对接/api/group-buy/initiate和/api/group-buy/join。

### 4. AI食材助手页（pages/ai-food/ai-food.vue）

#### 核心需求（课题特色功能）

- 输入框：支持输入食材名称/用餐场景（如“番茄+鸡蛋 做法”）；
- 智能推荐：点击按钮返回模拟的搭配建议、做法步骤；
- 结果展示：卡片式布局，含标题、步骤列表、配图，支持收藏。

#### 技术实现要求

- 输入框适配小程序，按钮用主色#F08800；
- 推荐结果：前端模拟关键词匹配，联调后对接/api/ai-food/recommend。

### 5. 其他页面（基础版，毕设保底功能）

| 页面                  | 核心需求                                                        |
| --------------------- | --------------------------------------------------------------- |
| 订单页（order）       | 展示订单列表（待成团/已成团/已取货），支持筛选、查看详情        |
| 团长页（leader）      | 展示团长拼团数、佣金统计、待审核订单（仅前端模拟数据）          |

## 五、样式规范（严格对齐课题视觉）

### 1. 主题色（不可修改）

- 主色：#F08800（导航栏、按钮、价格、tabBar选中态）；
- 辅助色：#E8F5E9（商品卡片背景、轻提示背景）；
- 强调色：#2F5233（AI助手模块、重要文字）；
- 中性色：#F8F8F8（页面背景）、#666666（普通文字）、#333333（标题文字）。

### 2. Tailwind使用规则

- 单位：全部用预配置的rpx映射（如px-4=16rpx、text-base=28rpx）；
- 禁止：手写大量原生CSS，仅补充Tailwind覆盖不到的样式（如文字截断）；
- 复用：通用样式抽离为Tailwind自定义类（如在uni.scss中定义`.card { @apply bg-white rounded-lg p-4; }`）。

### 3. 小程序适配规则

- 图片：全部用image组件，设置mode="aspectFill"，避免变形；
- 交互：禁用hover伪类，用active-class实现点击反馈；
- 布局：避免使用flex-wrap: wrap以外的复杂布局，适配小程序不同机型。

## 六、接口联调规范（对接后端）

### 1. 接口基地址

```ts
// src/api/index.ts
const baseUrl =
  process.env.NODE_ENV === 'development'
    ? 'http://localhost:3000/api' // 本地开发
    : 'http://[你的演示地址]/api' // 毕设演示（可选）
```

### 2. 统一请求格式

- 请求拦截：添加token（毕设模拟为固定值）、设置Content-Type: application/json；
- 响应拦截：统一处理code码（200=成功，400=参数错误，500=服务器错误），弹窗提示错误信息；
- 示例（商品列表接口）：

```ts
// src/api/goods.ts
import request from './index'
export const getGoodsList = (params: { category?: string; sort?: string }) => {
  return request({
    url: '/goods',
    method: 'GET',
    params
  })
}
```

## 七、毕设适配关键要点

1. **数据模拟**：联调前所有页面用前端模拟数据，确保无后端也能演示核心功能；
2. **演示适配**：微信开发者工具中设置“不校验合法域名”，本地启动后端即可完成演示；
3. **简化点**：跳过微信真实授权（固定user_id=1）、无需对接支付、AI推荐仅模拟关键词匹配；
4. **代码规范**：变量/函数用小驼峰，组件名用大驼峰，注释覆盖率≥30%（毕设文档易展示）；
5. **性能适配**：商品列表用懒加载，图片用压缩后的占位图，避免演示时卡顿。
